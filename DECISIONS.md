# Decisions — danulrich-site

_Real decisions only: what we chose, why, and what lost. Newest on top. Claude writes it; Dan reads it. (Started 2026-07-07.)_

- **2026-07-23 — Astro upgraded 5→7 (branch `astro-7-upgrade`); Tailwind stays on v3, now wired through PostCSS.** Why: Dan approved the migration, and it clears the last audit findings (0 vulnerabilities after); the retired @astrojs/tailwind integration was replaced with plain PostCSS wiring so the styling output stays byte-identical — one major change at a time on a live site. Rejected: jumping to Tailwind 4 in the same pass (bigger restyle risk, can be its own project later).
- **2026-07-22 — Astro stays on v5 for now; the 5→7 upgrade is a separate, Dan-approved migration.** *(superseded 2026-07-23)* Why: the only unpatched audit findings live in Astro's own bundled image library, which runs at build time only (zero exposure to site visitors), while a two-major-version jump risks breaking the live site. Rejected: `npm audit fix --force`, which would have jumped to Astro 7 unreviewed.
- **2026-07-22 — Self-healing security added: a pre-commit guard (`.githooks/pre-commit`) that blocks secrets/phone/personal-email from ever being committed, plus Dependabot weekly dependency PRs.** Why: this repo is public, so "no PII/secrets in git" should be enforced by machine, not memory; and security patches should arrive as review-and-merge PRs instead of relying on someone remembering to audit.
- **2026-07-22 — npm is the repo's one package manager; the template's leftover `pnpm-lock.yaml` was deleted.** Why: two lockfiles drift apart and confuse both Dependabot and future installs; only npm is actually installed and used on this machine.
- **2026-07-22 — Model-choice guide baked into CLAUDE.md: Sonnet for mechanical edits, Opus 4.8 (+plan mode) for design, security, and migrations.** Why: matching model to stakes saves quota on routine work and buys quality where mistakes are outward-facing; the table makes it a standing rule instead of a per-session judgment call.
- **2026-07-07 — "Download PDF" serves a WEB-SAFE variant: contact line = location + LinkedIn, no phone/email.** Why: this repo and the site are public — contact PII would be scraper food and a permanent git record. The full-contact resume stays a direct-send file from career-ops.
- **2026-07-06 — Phase 2 resume built on a branch; nothing reaches production without Dan's review of the Access-locked preview.** Why: every content change to a public career property is outward-facing; the branch+preview flow makes "review before live" structural instead of a promise.
- **2026-07-06 — Resume data model: ONE canonical wording per bullet with a `lanes[]` list, instead of per-lane rewrites.** Why: four parallel phrasings per bullet is exactly the drift risk the one-source-of-truth rule exists to prevent; lane filters dim, they don't rewrite.
- **2026-07-06 — Base theme = Astro Nano; fonts = Source Serif 4 (headings) + Inter (body), self-hosted; interaction = native HTML (`<details>`, Popover) + one small vanilla-JS island; diagrams later = build-time SVG (D2/hand-authored), not Mermaid.** Why: typography-first executive credibility with near-zero JavaScript — every dependency must justify itself. Rejected: heavier themes (Astrofy/DaisyUI = generic dev-template look) and runtime diagram libraries.
- **2026-07-06 — Preview deployments locked behind Cloudflare Access; `_headers` ships noindex for pages.dev hosts only.** Why: preview URLs otherwise stay public forever, and raw pages.dev copies would compete with danulrich.com in search. The real domain stays fully indexable.
